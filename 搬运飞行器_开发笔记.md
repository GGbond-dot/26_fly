## 1. 题目（2026 杭电电赛 X 题 搬运飞行器）

- 场地：A、B 为 50×50cm 对角起停区；场内 4 个边长约 15cm、高 50~150cm、顶面白色的高台。
  其中 **3 个高台上各放一片黑色圆铁片（直径约 5 / 8 / 11cm）**，1 个高台为空。
- 高台中心距边界 ≥50cm，高台间距 ≥80cm。
- **基础要求**：随机指定 A 或 B 起飞 → 一键起飞 → 降落到对角起停区；记录并显示起飞到降落的时间。
- **发挥要求**：起飞 → 自主找黑铁片（找到滞空 3s + 声光提示）→ 搬到那个空高台，
  **大的在下、小的在上**叠放 → 完成后降落到对角起停区。
- 启动后不得人为干预。

## 2. 场地坐标与约定（已写死进代码）

- 起飞点 = map 原点 (0,0)，扫描终点 (scan_end_x, 0) = (250, 0) cm。
- 降落点（对角 B）= (landing_x, landing_y) = (250, -250) cm。
- **空高台 = 离 B 最近的那根**（场地我们自己布置，已固定这个约定）。代码用
  `nearestToLanding()` 选它作放置目标。
- 检测有效 bbox（map 系，米）：x∈[0.25,2.25], y∈[-2.25,-0.25]（见 pillar_detector launch/param）。
- **实际横截面**：空柱（放置目标）= **30×30cm**，顶面宽容、叠放/落点余量大；
  其余载片柱 = **比 15cm 略大**，是取物对准较紧的一环（arm_offset 已机械共轴=0，靠视觉对中心要稳）。
- **所有柱顶都有黑色方边框**（空柱也有，只是没铁片）→ 视觉对所有柱子都能锁边框对中心；
  占比 = 圆/边框框面积，空柱采不到圆 = 占比缺失 = 空柱信号。

## 3. 硬件 / 传感器设计（关键，别改错）

### 两个激光的分工（重要结论）
- **面阵激光**（`laser_array_pkg`，节点 `laser_array_ground_node`，串口 /dev/ttyS3）：
  16 束，取 `max(valid)` 当地面 + 双峰检测 + 全遮挡冻结（`max_hold_frames` 帧）。
  **设计上抗突变、只认地面、刻意无视柱子** → 用作**飞控的 z 高度反馈**。
  话题 `/laser_array/ground_height`（Int16, cm）。
- **点阵激光**（单点，经 STM32 回传，uart_to_stm32 把它发到 `/height`，Int16, cm）：
  打到柱顶会有清晰阶跃 → 用作**测柱子高度**。
  ⚠️ **2026-05-22：点阵拉近相机到 2.5cm，第二趟伸臂会挡点阵 → `/height` 只能第一趟用**
  （机械臂收起、点阵无遮挡）。第二趟下降禁用 `/height`，只用面阵。
- **测柱高公式**（`tryStepHeight`，**第一趟 SURVEY 的 MEASURE_HEIGHT 测，逐柱含空柱，存进 survey_results_[i].height_cm**）：
  `drop = (面阵地面读数 - LASER_MOUNT_DIFF_CM) - 点阵读数`；
  **窗口内累计所有 drop≥`sample_pillar_drop_thresh_cm`(20) 的"命中帧"，攒够 `height_min_hit_frames`(默认5) 帧
  即取中位数为柱高，不要求连续**（2026-05-22 改，见下方"测高判定改版"）。
  早先是"连续3帧突变且帧间差≤5"，但窄柱顶激光反复滑进滑出、命中帧零散，连续条件难满足→漏测。
- **为什么用占比判铁片大小而不是直径**：柱高不统一→直径像素随高度变；
  而"圆面积/方框面积"占比与高度无关，所以用 `/circle_area_ratio` 排大小。

### 关于"面阵停在柱顶会不会突变"的结论
- 面阵冻结只在 **16 束全打在柱子上（无任何束见地面）** 时触发，且只撑 `max_hold_frames`(默认20≈0.4s)。
- 只要还有任意一束见地面 → `max()` 咬住真地面，**不冻结不漂移**。
- 点阵与面阵物理间距 ~20cm，对柱子正上方时面阵阵列大概率仍扫到旁边地面 →
  现实几何下"全遮挡"基本不发生。**先预设面阵不被遮挡**，出问题再处理。
- z 反馈全程留在面阵（飞控内部要吃稳定高度，不能突变，这是底线）。

### 执行机构
- 舵机机械臂：`/servo_control`（UInt8，0x01 伸出/放下 / 0x00 收起，串口帧 0x11）。
- 电磁铁：`/electromagnet_control`（UInt8，0x01 吸 / 0x00 松，串口帧 0x33）。已从 GPIO 迁到 STM32。
- 声光提示（蜂鸣器+LED）：`/buzzer_led_control`（UInt8，0x01 开 / 0x00 关，串口帧 **0x22**，蜂鸣器与 LED 绑同一帧）。
  硬件：PD15→蜂鸣器，PD14→WS2812B LED DIN。上电 uart 与 mission 各发一次 0x00 关闭。

### 飞控固件侧协议（2026-05-21 已与下位机核对，上位机无需改动）
匿名帧格式：`0xAA | addr | ID | dataLen | data... | sc1 | sc2`；
`sc1=前(dataLen+4)字节累加和`，`sc2=sc1 逐次累加和`（serial_comm 的 build/checksum 与之一字不差）。
- **0x11 舵机**：`data[0]==1 → TIM4_CH1(PD12)=1260 舵机下/抓取位`；其它 `→500 收起`。
- **0x33 电磁铁**：`data[0]==1 → PD13 置位(吸)`；其它 `→ 复位(松)`。
- **0x07 面阵地面高度（上位机→飞控）**：2B int16 LE cm → `rosData.laser_height_cm`，
  飞控再封 **0x34 距离帧(direction=0,angle_100=270,distance=该高度)** 经 UART5 喂 IMU 作 z 反馈。
- **0x05 高度（飞控→上位机→`/height`）**：固件外发的是 `ano_of.of_alt_cm`（激光测高模块/光流口的对地单点高度），
  **实测打到柱顶有清晰阶跃**，第一趟 `tryStepHeight` 测柱高靠它（早先笔记说的"点阵单点"即此路）。
  注意：这条已不是 `fc_alt.st_data.add`。⚠️ 第二趟伸臂会挡它 → 第二趟不订阅/不用这条测高。

## 4. 节点 / 话题数据流

```
pillar_detector_tf  --(/detected_pillars, 仅xy)-->  mission
visual_node         --(/fine_data 像素偏差, /circle_area_ratio 占比)--> mission
laser_array_ground  --(/laser_array/ground_height 面阵)--> mission & uart(→飞控帧0x07)
uart_to_stm32       --(/height 点阵, 来自STM32帧0x05)--> mission
mission(pillar_pickup_mission)
   --(/target_position [x,y,z,yaw])--> PID --(/target_velocity)--> uart --> STM32(飞控)
   --(/visual_takeover, /active_controller, /servo_control, /electromagnet_control)
   --(/pillar_detect_enable)--> detector
   --(/route_choice=1)--> uart (打开 /target_velocity 转发，否则不下发)
   --(/mission_complete)--> uart (帧发3次, 关闭转发)
```

- **检测本身不直接发飞控指令**；检测只给 xy。mission 收到后切目标点，经 PID→uart→飞控。
- uart 必须先收到 `/route_choice ∈{1,2,3}` 才转发速度；mission 启动时连发 10 次 `/route_choice=1`。

## 5. 本轮已实现：两趟任务逻辑

文件：`src/activity_control_pkg/{include/activity_control_pkg/pillar_pickup_mission_node.hpp, src/pillar_pickup_mission_node.cpp}`
启动：`src/my_launch/launch/demo3.launch.py` → 含 `pillar_pickup_mission.launch.py`

**关键设计（本轮重点）：摄像头对准的是"画面中心"，不是激光/机械臂。**
摄像头、点阵激光、机械臂吸取点在机体上是三个不同位置，所以分两个偏置、各站各的位：
- `cam_offset`（相机→点阵激光）：**测高那步**用，把激光挪到柱心。
- `arm_offset`（相机→机械臂吸取点）：**下降取物/放置**用，把吸取点挪到铁片心。
吸取后机械臂提起，不遮挡摄像头，放置时空柱照样能视觉"对中心"（空柱高已在第一趟测好，放置不再测高）。

状态机：
```
SCAN (起飞(0,0,flight_h) → 扫描终点(250,0)，到起飞点开检测、到终点关检测)
 → WAIT_PILLARS (等 /detected_pillars 聚类结果, 超时则直接降落)
 → SURVEY  第一趟【测占比 + 测柱高】，逐柱（就近遍历，含空柱）：
      APPROACH(飞到柱上方 pillar_visit_height)
      CENTER(接管对准边框/铁片中心 + 采集 /circle_area_ratio 占比;
             找到铁片→开声光滞空 survey_signal_hold_sec(3s)→关声光)
      MEASURE_HEIGHT(关接管, +cam_offset(-2.5)把点阵挪到柱心, 累计命中帧测柱高存储;
                     超时→该柱高度未知, 第二趟跳过/空柱不能放)
      记录每柱 {plate_ratio, has_plate, height_cm, has_height}
 → planPickupOrder(): 排除空柱和无片柱，按 plate_ratio 降序 = 大→小
 → PICKUP  第二趟，按大→小逐片【用第一趟柱高, 只用面阵, 不碰 /height】：
      APPROACH → CENTER(接管对准铁片中心 xy; 无 has_height 则跳过此片)
       → MEASURE_HEIGHT_GRAB(抓取前现场测高，超时回退第一趟柱高) → startGrabDescend()
       → 【分段下降，2026-05-23】DESCEND_MID(降到下一中停读数, 进入即开电磁铁)
           ⇄ RECENTER_MID(中停二次对准方框中心, 对上‖超时往下) … 循环到无中停点
       → DESCEND_FINAL(末段盲降 z=R+柱高−grab_press)
       → HOVER_GRAB(先磁后臂) → CLIMB_BACK
       → OBSERVE_GRAB(靠占比判成败=爬升后还看到圆[非NaN]即铁片还在=失败；
           最小片先到巡航高度下方20cm观察，成功后再爬回巡航高度继续放置，失败则从该低高度继续下降重试；
           普通片失败重试≤pickup_max_attempts 回 MEASURE_HEIGHT_GRAB(重测高+重建分段)；单片用尽→**跳下一片**(pickup_iter++、attempts清零);
           全部片各自试满都失败才 startLanding; 2026-05-22 修，原来单片失败即降落)
       → GOTO_DROP(空柱; 空柱高未知则带片降落)
       → CENTER_DROP(接管只对空柱 xy, 高度用第一趟测好的)
       → DESCEND_DROP(z=R+空柱高+stack_count×厚度+drop_gap)
       → HOVER_DROP(伸臂+松) → CLIMB_AFTER_DROP, stack_count++
 → LAND (对角起停区 land_approach→land_hover→land)
 → DONE (/mission_complete)
```
（R = ARM_GROUND_AREA_CM = 机械臂伸直触地时面阵读数；z 全是面阵目标读数，见第6节）

要点：
- 大下小上 = 占比大的先抓先放垫底。
- 叠放每多一层落点抬高 `plate_thickness_cm`。
- 第一趟 `MEASURE_HEIGHT` 先 settle 到激光位(柱心)再采样，避免移动途中扫掠污染。

### 16.2 视觉"中心框优先"：压掉起停区 A 方框抢锁（疑似第14节占比错乱真因）
- 现象：对齐**离起点最近的柱①**时，左下角**起停区 A**（粗黑线围的≈50cm 角区方框 + 里面空心粗线字母 A）

  A 框 50cm 远大于柱顶黑框（柱①>15cm）→ 一挤进画面 score 就压过柱框，`StableRectangleTracker` 锁到 A。
  demo3 是 `center_source=square`，对准信号 `/fine_data` 跟锁定框中心 → 飞机往 A 偏；

- 为什么不"解码 A"：`/apriltag_code` 是假的（visual_node.py:108-111 把静态参数原样发，demo3 设 -1），

- 改法：方框打分加**画面中心优先**因子——离画面中心越近分越高，把边角的 A/B 大框压下去。
  飞机在柱①正上方对齐时柱框在画面**正中**、A 框在**边角**，权重一压 A 出局；锁不到 A 则圆只在柱框里找，
  占比/对准/抓取顺序一起恢复。对所有柱子受益（永远优先正下方那个框）。配合 tracker `switch_required_hits=3` 迟滞更稳。
  - 公式：`score *= center_score**rect_center_bias`，`center_score=1−中心距/半对角线`（正中1→画面角0）。
    `rect_center_bias` 默认 **2.0**；0=关闭(纯面积旧行为)；试飞嫌还抢锁就加大(3~4)。
  - 风险低：CENTER 阶段飞机已 `isReached` 到柱① xy（容差9cm），柱框本就贴近画面中心；A≥50cm 开外离中心远。
  - **不碰圆检测逻辑**（如试飞发现万一被填实的 A 仍漏过，再叠"圆度/solidity 收紧"做第二道防线）。

### 背景：当前放置流程（第15节定的，放置侧"不加对准"）
`OBSERVE_GRAB`(抓成功) → `GOTO_DROP`(飞空柱上方) → `CENTER_DROP`(视觉对空柱框中心，仅 xy) →
`DESCEND_DROP`(**单段盲降** z=R+空柱高+已叠+drop_gap) → `HOVER_DROP`(伸臂+松磁) → `CLIMB_AFTER_DROP`。
本节把放置下降也升级成"分段中停对准"，并修 `HOVER_DROP` 的时序 bug。

### 18.1 放置释放时序修复（核心 bug）
- **现象**：`HOVER_DROP` 进入时 `publishServo(true)` 和 `publishMagnet(false)` **同一时刻发**（cpp `enterPickupSub` 的 `HOVER_DROP` case，line ~991-999）。
  机械臂从收起到伸直需要 **~1.2s**，现在臂还没伸直磁就松了 → 铁片提前脱落、放不准。
- **改法**：把放置释放拆成时间序列（机械臂动作与松磁解耦）：
  1. 进入 `HOVER_DROP`：只 `publishServo(true)`（开始伸臂），记 `sub_hover_start_`，**不再同帧松磁**。
  2. `stepPickup` 的 `HOVER_DROP` case 按 `elapsed=(now−sub_hover_start_)` 分两个时刻触发（用一个 bool `drop_released_` 防重复发磁）：
     - `elapsed >= arm_extend_sec_`(1.2) 且未释放 → `publishMagnet(false)`，置 `drop_released_=true`（此刻臂已伸直，片贴到叠面再松）；
     - `elapsed >= arm_extend_sec_ + drop_settle_sec_`(1.2+0.5) → `publishServo(false)` 收臂，`++stack_count_`，进 `CLIMB_AFTER_DROP`。
- **抓取侧 `HOVER_GRAB` 不用改**：当前先磁后臂、`hover_grab_sec_`=2.0s 收臂，2.0>1.2 已覆盖伸臂耗时，且抓取是"先充磁再接触"不存在提前松的问题。
- **新参数**：`arm_extend_sec`(1.2, 机械臂伸直耗时)、`drop_settle_sec`(0.5, 松磁后片落稳再收臂)。新成员 `bool drop_released_`。

### 18.2 放置落点多压 2cm
- **需求**：放到空柱（含已叠片）上时再多下降 2cm，确保片压到叠面再松磁。**这是放空柱的下降，不是最终降地面**。
- **改法**：放置落点 z 公式加一项下压：`z_drop = R + 空柱高 + stack_count×plate_thickness + drop_gap − drop_press`。
  改 `DESCEND_DROP`（或下方 18.3 改造后的放置末段）算 z 处。
- **新参数**：`drop_press_cm`(2.0)。约定同 grab_press：值越大压得越低；放歪/压坏→回调，够不到→加大。

### 18.3 放置也走多段中停对准（复用抓取机制）
- **依据（已查视觉代码确认）**：demo3 `center_source=square`（visual_node.py:200），`/fine_data` 走**黑方框 4 角几何中心**，
  与圆无关；圆只供 `/circle_area_ratio`。所以**空柱顶只有黑框、没有圆，对准信号照常输出**，第一片放置无对准问题。
  且铁片吸在电磁铁下方**不挡**摄像头，只有机械臂伸直才挡——而放置下降全程臂是收起的 → 下降全程可视觉对框中心。
- **改法（推荐：标志位复用，不另写平行状态机）**：加成员 `bool descend_is_drop_`。
  - 新增 `startDropDescend()`：仿 `startGrabDescend()`，置 `descend_is_drop_=true`，调 `buildDescendPlan(空柱高, /*is_drop=*/true)` 建中停表，
    有中停点→进 `DESCEND_MID`，否则直接 `DESCEND_FINAL`。`startGrabDescend()` 里置 `descend_is_drop_=false`。
  - `CENTER_DROP` 对准完成后调 `startDropDescend()`（替换原来直接进 `DESCEND_DROP`）。
  - 复用的 `DESCEND_MID`/`RECENTER_MID`/`DESCEND_FINAL` 三个子状态内按 `descend_is_drop_` 切换：
    - **xy 来源**：抓取用 `survey_results_[si].x/y`（载片柱）；放置用 `empty_pillar_x/y_cm_`。
    - **末段 z 公式**：抓取 `R+柱高−grab_press`；放置 `R+空柱高+stack×厚度+drop_gap−drop_press`(见18.2)。
    - **y 偏置**：抓取末段加 `grab_final_dy_cm`(见18.5)；**放置不加**（放置是松磁释放、要对准已叠片正中，加偏置反而叠歪）。
    - **末段到位后去向**：抓取 `DESCEND_FINAL`→`HOVER_GRAB`；放置→`HOVER_DROP`。`RECENTER_MID` 递增 ckpt 的逻辑两者相同。
    - **下降安全底线**（第17节 `stepPickup` 开头 guard）：把 `in_grab_descend` 判断加 `&& !descend_is_drop_`，**放置下降不走安全底线**
      （放置 z=R+空柱高(≥50)+… ≥77，绝不会低于 22；且 abort 分支的"回 MEASURE_HEIGHT_GRAB 重试"是抓取语义，放置不能复用）。
  - `buildDescendPlan(height, is_drop)` 加 `is_drop` 形参：is_drop 时 `grab_reading` 用放置 z 公式、且 **seg 固定 30 不加密**（见18.4）。
  - 原 `DESCEND_DROP` 子状态可保留枚举但不再用（或直接删，改走 DESCEND_MID/FINAL 复用路径）；`descend_seg_len`/`recenter_timeout` 复用抓取同值。
- **⚠️ 实飞风险（叠片对框检测）**：叠第 2、3 片时柱顶已有黑铁片，黑框内"洞"从纯白变白环。
  空柱框(~30cm)≫最大片(11cm)，白环面积通常仍够、`hole_ratio≥min_hole_ratio_for_border`(0.08, detector_core.py:27/497) 仍成立 → 大概率仍锁框；
  万一被填到 hole_ratio<0.08 框丢，visual_node 发 `[0,0]`(visual_node.py:132/233)→mission 当"已对准"往下降，空柱余量大能接住、RECENTER 超时兜底，不卡不摔。叠片那两次盯一下框检测即可，代码先不为此改视觉。

### 18.4 抓最小片分段加密（seg 20）
- **需求**：抓占比最小那片（叠在最顶、最易歪）时分段更密、多对准一次；其余片不变；**放置一律不加密**（平台大怎么放都掉不下去）。
- **改法**：`buildDescendPlan` 算 seg 时：抓取且 `pickup_iter_ == pickup_order_.size()−1`（末位=占比最小）→ 用 `descend_seg_len_min_plate_cm`(20)，
  否则用 `descend_seg_len_cm`(30)；is_drop 恒用 30。
- **新参数**：`descend_seg_len_min_plate_cm`(20)。

### 18.5 抓取末段 y 负向偏置 −2cm
- **需求**：最后对准/下降时飞机再往 map **y 负方向 −2cm**，电磁铁吸得更准（吸取点相对相机中心的残余偏置）。
- **改法**：加参数 `grab_final_dy_cm`(−2.0)，**只在抓取的 `DESCEND_FINAL`** 落点 xy（`applyArmOffsetToTarget` 之后）再叠加 `ty += grab_final_dy_cm_`。
  放在**盲降段**而不是 RECENTER 视觉对准段——视觉接管时是把画面拉向中心(0,0)、塞不进硬偏置；让视觉先对正框中心，最后盲降平移 −2 再扎下去吸。
  放置末段不加（见18.3）。轴定义见第12节：map +y ↔ 画面正左方。若实飞发现偏反向，把符号取正。

### 18.7 下一轮实验待办：解耦分段视觉微调 + 改放置释放高度

#### 抓取侧：做 A/B 对比，不要默认相信分段微调
用户原话要点：**分段视觉微调不一定很管效，可以给代码做好解耦，解耦成视觉微调到中心之后就直接下降抓取，不做分段，然后做对比实验，看看哪个效果更好。**

建议下一轮把抓取下降模式做成参数，而不是继续硬绑分段：
- `grab_descend_mode = segmented`：当前方案，`DESCEND_MID → RECENTER_MID → DESCEND_FINAL`。
- `grab_descend_mode = direct_after_center`：`CENTER` 视觉对准到中心后，直接 `DESCEND_FINAL` 到抓取高度，不做中停分段视觉微调。
- 对比指标：抓取成功率、是否抖、是否被中停视觉带偏、第三小片是否需要分段。
- 初步判断：当前已经出现“没分段也能抓准”的现象，说明分段微调可能不是主因，甚至可能引入额外扰动；必须用参数化 A/B 实飞验证。

#### 放置侧：不要完全贴到空柱再松磁，改成上方释放
用户原话要点：**放的时候不要完全贴到空柱再松开电磁铁。建议在现在代码基础上，先对准一次中心，然后直接降落到空柱上方 10cm 左右，伸下机械臂，松开磁铁，然后悬停 1s，免得惯性影响。**

建议下一轮把放置释放也参数化：
- 新参数建议：
  - `drop_release_clearance_cm = 10.0`：放置末段目标不再是贴到叠面，而是叠面上方约 10cm。
  - `drop_post_release_hover_sec = 1.0`：松磁后原地悬停 1s，避免机体惯性/机械臂动作把片带偏。
  - `drop_press_cm` 可临时设回 `0.0` 或废弃为兼容参数；若采用上方释放，公式应改为 `R + 空柱高 + stack×厚度 + drop_release_clearance_cm`。
- 推荐流程：
  1. `CENTER_DROP` 视觉确认空柱中心并更新 anchor。
  2. 关闭视觉接管，直接下降到 `空柱叠面上方约 10cm`。
  3. 伸机械臂。
  4. 松磁。
  5. 悬停 `1s`。
  6. 收臂并爬升。
- 这个方案比“完全贴到空柱/叠片再松磁”更少受接触摩擦、下压、机体惯性影响；尤其第二、第三片放置时，视觉黑框已经可能被铁片干扰，越不应该在贴近状态反复追目标。

#### 下一轮建议优先级
1. 先保留当前 `drop anchor` 逻辑，因为它解决“视觉微调结果被后续目标发布抵消”的硬问题。
2. 再加 `grab_descend_mode` 做抓取分段 vs 直接下降对比。
3. 再把放置末段改为“空柱上方 10cm 释放 + 悬停 1s”，这可能比继续调 `drop_press_cm` 更符合现场现象。

### 18.9 ✅ 已实现：精准降落（视觉对准 B 黑框中心，2026-05-23，待编译机 colcon build）
> 需求：题目降落点 B（对角起停区）外有黑色方框，要对着 B 框中心比较精准地降落。

#### 依据（不用改视觉）
- B（含 A）起停区是"粗黑线围的 ≈50cm 角区方框"，**本来就会被 `visual_node` 的 `detect_black_rectangle_border` 识别成黑框**
  （见 16.2：当初是为对柱子时不锁 A/B 才加 `rect_center_bias` 压制；现在降落要锁 B，正好顺势复用）。
- demo3 `center_source=square`，`/fine_data` = 黑框 4 角几何中心；视觉接管 PID 会把它拉到画面中心。
- 飞到 B 上方 `land_align_height`(默认 150) 时：B 框居中、对角的 A 框在 FOV 外，`rect_center_bias=2.0` 优先锁正下方 B。

#### 改法（新增独立降落子状态机，复用对准机制；含一次中停二次对准）
- 新枚举 `LandSub{APPROACH, CENTER, DESCEND_MID, RECENTER, DESCEND}` + `enterLandSub`/`stepLanding`：
  - `APPROACH`：飞到 B 名义坐标上方 `land_align_height`（粗定位，takeover off）。
  - `CENTER`：开视觉接管对准 B 框中心；`isVisuallyAligned()` 或 `visual_align1_timeout` 超时即结束。
    对上时机体就在框中心正上方 → 记当前实际 map 位姿为 `land_anchor`（同 drop anchor 思路）；超时则回退名义 B 坐标。
  - `DESCEND_MID`：关接管，从对准高度再下降 `land_recenter_drop_cm`(默认60) 到中停高度（位置保持在 anchor）。
  - `RECENTER`：中停高度（更近、像素更准）开接管**再对准一次** B 框（`visual_align2_timeout` 超时则沿用上次 anchor），对上刷新 anchor。
  - `DESCEND`：关接管，位置保持在 anchor 正上方竖直降到 `land_height`，到位即发 `/mission_complete`、`active_controller=3`、进 DONE。
  - 中停保护：`land_recenter_drop_cm<=0` 或中停高度 ≤ `land_height` 时跳过 DESCEND_MID/RECENTER，CENTER 后直接 DESCEND。
- `startLanding()` 按 `land_visual_enable` 分流：true → 走上面状态机；false → 旧的纯位置三航点降落（兜底）。
- 主循环 `MissionPhase::LAND` 分流到 `stepLanding`（视觉）或 `stepWaypoints`（纯位置）。
- 设计：**降一段校准一次**（共两次对准：高空粗对 + 降 60cm 后近距精对），近地最后一段竖直降不再追（低空框胀出视野、fine_data 噪声大）。

#### 新参数
| 参数 | 默认 | 含义 |
|---|---:|---|
| `land_visual_enable` | `true` | 降落是否先视觉对准 B 框；false=纯位置降落（旧行为，B 框识别不稳时回退） |
| `land_align_height_cm` | `150`(=巡航) | 第一次对准 B 框的悬停高度。50cm 框居中可见、A 框在视野外；想更精就降低，但太低框胀出视野认不全 4 角 |

> 顺带把降落视觉按用户要求**单独开成一个模式**，且不走“解码字母 B”那条路（属伪需求，详见下）。

##### 根因（三层，按致命度）
1. **`[0,0]=已对准` 的致命误判（最关键）**：`visual_node` 框丢时 `/fine_data` 发 `[0,0]`，mission `recordFineData`
   无条件塞进 `fine_hist_`，`isVisuallyAligned()` 看 `[0,0]` 在像素阈值内 → 判“已对准” → 把当时带漂移的名义 B 位置记成 `land_anchor` → 竖直降下去自然偏。这个约定本是给 30cm 大空柱兜底的，降落对角区时是雷。
2. **B 框比柱顶框难认**：B 在场地角上、两边和场地外边界粗黑线连成一片，`findContours` 难抠出干净 4 角闭合方框；
   里面有字母 B 搅乱“洞”；高度一降 50cm 框胀出画面、丢角 → 不再是 4 角凸四边形 → 检测器直接放弃。和柱顶 15~30cm 干净带洞方框是不同场景，却在用同一个 `detect_black_rectangle_border` + 同一套参数。
3. **为什么不“解码字母 B”**：靠**位置**飞到 B 角，对角 A 在视野外，那个角没有第二个框要区分 → 认身份是伪需求、零价值；
   真正缺的是①稳定给框中心像素偏差②别把丢框当对准。OCR 不解决稳定性，纯烧算力，方向就不对。

##### 改法（C+A+B 三件，用户拍板全做）
- **(C) 堵假对准（最便宜、收益最大）**：`/fine_data` 加**第三位 valid**(1有效/0丢框)。
  `recordFineData(dx,dy,valid)`：**降落阶段 valid=0 直接不计入 `fine_hist_`/不刷时间戳** → 持续丢框时 `isVisuallyAligned` 攒不满
  → `CENTER`/`RECENTER` 超时 → 回退名义 B 坐标（已有逻辑），绝不拿漂移位置记 anchor。其余阶段（柱子/放置）**行为不变**（放置仍靠 `[0,0]` 兜底，故只在 `phase_==LAND` 才拦 valid=0）。旧版 visual_node 只发两位 → 默认按有效处理，向后兼容；PID 侧只读前两位也不受影响。
- **(A) 降落视觉单独成模式**：mission 进降落发 `/vision_mode="land"`（锁存 transient_local），`visual_node` 切到
  `_process_landing()` 专用分支：**只找 B 框中心、关圆检测（占比发 NaN）**，用一套**独立的 `landing_*` 参数**，和柱子那套彻底解耦——以后调降落不动柱子检测。其余时间 `"pillar"`（默认，行为不变）。
- **(B) 大框/出框兜底**：新增 `detector_core.detect_landing_box()`：①优先中心优先的 4 角凸方框（框完整在画面=高空记 anchor 时刻最准）；
  框 bbox 最长边 > `landing_oversize_frac`(0.72)×画面短边 = 太大/出框 → ②裁掉四周 `landing_roi_margin_frac`(0.12) 边带（排除沿画面边缘的场地外边界线）取中央 ROI 黑像素质心兜底；中央黑占比 < `landing_min_black_frac`(0.015) 判无目标返回 None(空画面不乱报中心)。
- 保留 18.9 的两段式 anchor 降落（高空粗对→记 anchor→降 60→近距精对→竖直降），低空那段仍纯位置不追框。

### 19.1 这趟实况（2 片，与现场吻合）
- 抓取顺序：#1 柱3@(76,-181) 占比0.443[最大片] survey高56；#2 柱0@(71,-82) 占比0.187[第二大] survey高**84**[最高柱]。空柱=柱2@(178,-161)高56。

- **第2片[柱0,最高 h84]：卡死，用户迫降。**

### 19.2 第二片卡死的四步级联根因
1. **现场测高把最高柱测低**：69 vs 真 84（分簇测高对窄高柱仍系统性偏低，点阵擦柱沿；survey 当时是对的 84）。
2. **一致性校验差 1cm 漏网**：判据 `(survey−现场) > live_height_consistency_cm(15)`；`84−69=正好15`，`15>15` 不成立 → **采用错值 69** → target z=`R27+69−grab_press`≈**89**（正确应 ~104，低 15cm）。
3. **真凶=面阵虚高**：下降到面阵≈97（临近 84 柱顶）后蹭柱**倾斜**，面阵取 `max(有效束)`、斜射打到很远斜地面 → **面阵虚高跳到 160~193 并持续约 20s**（221.8s→241.4s，`bimodal=1 obstacle=1`）。这虚高值是喂飞控的 z 反馈 → 飞控以为"我在 180、目标 89"→ **持续猛压下降 20s → 死压在柱子上**=用户看到的"卡住"。
4. **安全底线被蒙住**：`面阵<22→中止` 在这 20s 里面阵读 180≫22 **完全没触发**，直到面阵骤降 `102→76→44→20` 才在 20<22 触发——那时早已压柱/迫降。尝试2/3 面阵直接=10（已在地上=迫降后数据）。
- 一句话：矮柱整套 work、放置成功；最高柱同时踩中【测高偏低 + 一致性差1cm漏 + 倾斜致面阵虚高让安全底线失明】三坑叠加。

### 19.3 改动（用户拍板）
> 设计约束（用户强调）：**喂飞控/IMU 的 z 高度反馈必须连续、不可突变 → 面阵数据照常用、不滤、不改。**
> 所以不去"修正"虚高值，而是**纯统计判废**：面阵连续够多帧明显虚高，就认定这根柱子抓不到、直接放弃（不重试）。
1. **面阵虚高判废（核心）**：`areaHeightCallback` 累计连续"虚高异常"帧 `area_high_streak_`——
   面阵 > `pillar_visit_height_cm + descend_area_anomaly_margin_cm`(150+15=165) 即 +1，出现一帧正常即清零（不动喂飞控的值）。
   `stepPickup` 抓取下降段（DESCEND_MID/RECENTER_MID/DESCEND_FINAL，放置不算）守卫：`area_high_streak_ ≥ descend_area_anomaly_frames`(默认**10**≈5s@2Hz) →
   松磁收臂、发 `/pickup_failed`、`++pickup_iter` **跳下一片**（最后一片则降落）。放在安全底线 `<22` 判断**之前**。
   - **帧数要足够大以免误判**（那次虚高约 40 帧）；默认 10 远离误判、又比实际触发底线(20s 后)早约 15s。嫌误判调大 frames。
   - `startGrabDescend()` 每段下降开始清零 `area_high_streak_`，防上一柱残留。
2. **一致性阈值 15→8**：堵边界漏网（84−69=15 这种）。`>` 不变（阈值降到 8 后真实最高柱偏差~15 远超阈值，不再卡边界）。

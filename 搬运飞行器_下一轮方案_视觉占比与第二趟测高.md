## 1. 问题一：视觉占比错乱

### 1.1 涉及文件
- 正在用：`src/visual_pkg/visual_pkg/detector_core.py`（检测核心）+ `visual_pkg/visual_node.py`（ROS 节点，发 `/circle_area_ratio` `/fine_data`）。
- 调试工具：`src/dbh_visual_pkg/dbh_visual_pkg/opencv_init_v43.py`。
  **检测核心与 detector_core.py 几乎一字不差**，但它是带 OpenCV trackbar 调参面板 + 二值图调试窗口
  （primary/black/fused 三联）+ `classify_circle_size`(行17) 尺寸分类的单机工具。
  → **价值是离线/实时调参与诊断，不是拿来替换 detector_core**。改逻辑仍改 detector_core.py。

### 1.2 根因（已定位，函数 `detect_circle_in_square`，detector_core.py:545）

**空柱误检出假圆（虚高 0.237）**：
1. `effective_area = max(area, area_coeff*π·radius²)`（行 609-616）用 `minEnclosingCircle` 的半径把面积撑大。
   空柱白心里 adaptiveThreshold(INV) 会抠出阴影/反光/黑边内沿残渣（margin 只腐蚀 5px，厚黑边内沿残留），
   一段弧/一撮斑点被套上大外接圆 → `π r²` 虚增 → 占比虚高。
2. relaxed 兜底太宽（行 650：circularity≥0.34、solidity≥0.38），残渣弧线轻松通过。
3. **缺"必须是纯黑实心盘"判据**：现在跑在 fused 掩码（adaptive 边缘 OR 黑色）上，把边缘/暗处也当候选，
   丢了"实心黑"这个最强判据。空柱中心是白的，本就没有大块黑色实心区。

**最大盘被测成最小（0.023）**：
1. 上限直接 `continue`：`area > ratio_square_area*0.82`(行581)、`effective_area > ratio_square_area*0.88`(行613)。
   11cm 盘去掉黑边后在内白框里占比很高 → 撞上限被丢。
2. `contour_touches_boundary`（行584/625/657，strict+relaxed 都要求 `not touches_boundary`）：
   大盘边缘逼近黑边内沿 → 判触边 → 丢弃。
3. 真大圆被丢，只剩白框小噪声 → 占比塌成 0.023。

→ 两个故障方向相反（噪声被放大、真大盘被毙），所以排序/对准/空柱判定全乱。

### 1.3 改法（已定：**重构判据，回到"纯黑实心圆"**）

核心思路：铁片 = 一整块**纯黑实心圆盘**；空柱顶中心是白的、没有大块黑色实心区。把判据建立在这个本质上。

1. **从 HSV 黑掩码找铁片，而非 fused**：在 `detect_circle_in_square` 里对 ROI 用 `build_black_mask`
   的黑色掩码（而不是含 adaptive 边缘的 fused）找连通域。空柱白心在黑掩码里基本是空的
   （黑边已被 margin 排除）→ 占比≈0 → 自然判空柱。
2. **去掉/限制 `effective_area` 外接圆放大**：占比改用真实 contour 填充面积（或填充盘面积），
   不要用 `π·radius²` 把噪声吹大。
3. **加"实心"判据**：solidity 要高（实心盘≈1，残渣弧线低）+ 一个**绝对黑面积下限**杜绝小斑点。
4. **放开大盘**：上限 0.82/0.88 提到 ~0.95，且改"截断不丢弃"；大盘触边判定放宽
   （大盘贴近黑边是正常的，不该因此被毙）。
5. **可选**：用 dbh 的 `classify_circle_size`(opencv_init_v43.py:17) 把占比映射到 小5/中8/大11，
   `planPickupOrder` 改成按尺寸分类排序，比纯 raw ratio 更抗绝对值误差（见 1.5）。

### 1.4 验证手段（dbh 调参面板）
- 用 `src/dbh_visual_pkg`（`ros2 run dbh_visual_pkg ...` 或直接跑 opencv_init_v43.py）开 trackbar 面板 +
  二值图三联窗（按 `b` 开 Binary Debug）。
- 对着实物/录像逐柱看：**空柱黑掩码内应≈空 → 无圆**；**大盘 → 大占比**；中小盘占比单调递减。
- 调 `black_v_max`/`black_s_max`（黑色阈值）、solidity/circularity、上限。调好后把参数+逻辑搬进 detector_core.py。

### 1.5 改完要回归的下游
- `planPickupOrder`（mission cpp，按 `plate_ratio` 降序）：占比修对后排序自动对。
  若改用 `classify_circle_size` 分类排序需同步改 mission（见第 4 节风险）。
- 空柱判定：`finalizePlateRatio` 采不到合格占比帧 = 空柱（mission cpp:396）。占比修对后空柱占比≈0，
  天然采不够帧 → 判空柱（仍叠加"离 B 最近"双保险，见笔记第2节）。

## 2. 问题二：第二趟抓取也现场测高

### 2.1 现状（笔记第3/5节 + 实读 cpp）
- 第一趟 `SURVEY/MEASURE_HEIGHT` 逐柱测高存 `survey_results_[i].height_cm`（cpp:689-711, `tryStepHeight` cpp:376）。
- 第二趟下降**写死**用第一趟柱高：
  - `DESCEND_MID`: `z = ARM_GROUND_AREA_CM + ph + mid_clearance`（cpp:801, `ph=survey_results_[si].height_cm`）
  - `DESCEND_FINAL`: `z = ARM_GROUND_AREA_CM + ph - grab_press`（cpp:812）
- 现在第二趟完全不订阅/不用 `/height`（因当时认为伸臂会挡点阵）。

### 2.2 用户澄清的新事实
**只有夹着铁片时才遮挡点阵测高模块**。臂收回/没夹片时点阵无遮挡 → 第二趟在没夹片时可以测高。
（HOVER_GRAB 结束会 `publishServo(false)` 收臂，cpp:950；抓失败后臂已收回、未夹片 → 点阵畅通。）

### 2.3 新流程（已定）
- 第一趟：照旧测高存 `survey_results_[].height_cm`（**不动**）。
- 第二趟抓取：**每次下降前、且没夹着铁片时**，现场测一次当前目标柱高，下降抓取用**新鲜测的高度**：
  - 首次抓：CENTER 对准后 → 现场测高 → DESCEND_MID/FINAL 用新高度。
  - 抓失败重试（attempts < max，臂已收回、未夹片）：**重试前再测一次** → 再下降。
- 判定**抓到了** → 飞空柱放置：**不再测高**（夹着片会挡点阵），放置 z 用第一趟测好的**空柱高**
  `empty_pillar_height_cm_`（cpp:867, `DESCEND_DROP`，**不动**）。
- 现场测高**超时/失败兜底**（已定）：**回退用第一趟 `survey_results_[si].height_cm`** 下降，不放弃这片。

### 2.4 抓取成功判定（用户强调 + 已定加固）
两种情况都 = 认不出圆 = NaN = **成功**，现有判定方向正确：
- (a) 铁片被吸走 → 柱顶变空 → 看不到圆（NaN）。
- (b) 铁片遮挡摄像头 → 看不到圆（NaN）。
OBSERVE 期间收到**非 NaN（还看到圆）= 铁片还在原位 = 失败**；NaN = 成功。
（回调 cpp:236-238 把 OBSERVE 期间任何非 NaN 置 `pickup_observed_plate_=true`；判定 cpp:961-1003。）

**加固（已定）**：
1. 成功判据复用问题一的"纯黑实心盘"——避免情况(a)柱顶变空后被假圆误判成失败（见 3.4 耦合）。
2. 要求**连续 N 帧看到真盘**才判失败，防单帧噪声/假圆把"成功"误判成"失败"
   （笔记第14节已列此为隐患：现在单帧非 NaN 即判失败、偏敏感）。
   实现：OBSERVE 期间累计"看到真盘"帧数，≥N 才置失败；NaN/非真盘帧不累计。

## 3. 第二趟测高的精确接入点（给新对话）

文件：`src/activity_control_pkg/{include/activity_control_pkg/pillar_pickup_mission_node.hpp, src/pillar_pickup_mission_node.cpp}`

### 3.1 新增子状态
`enum class PickupSub`（hpp:118-132）在 `CENTER` 与 `DESCEND_MID` 之间加一个：
```
MEASURE_HEIGHT_GRAB,   // 第二趟下降前现场测当前柱高（仅未夹片时；点阵无遮挡）
```
注意区分第一趟的 `SurveySub::MEASURE_HEIGHT`，命名别撞。

### 3.2 状态流改动（cpp `stepPickup` / `enterPickupSub`）
- `CENTER` 对准完成（cpp:918-936）：原来直接 `enterPickupSub(DESCEND_MID)`。
  改为 `enterPickupSub(MEASURE_HEIGHT_GRAB)`。
- 新增 `enterPickupSub(MEASURE_HEIGHT_GRAB)`：仿第一趟 `SurveySub::MEASURE_HEIGHT`(cpp:616-624)：
  - `applyCameraOffsetToTarget(tx,ty,0.0)` 把点阵移到柱心（cam_offset 现 -2.5）。
  - `publishVisualTakeover(false)`；发目标到 `pillar_visit_height_cm_`（或当前高度），settle。
  - `height_hit_samples_.clear();`。
- 新增 `stepPickup` 里 `case MEASURE_HEIGHT_GRAB`：仿 cpp:689-711：
  - `isReached(...) && tryStepHeight(h)` → 把 h 存进**本片现场高度缓存**（新增成员，
    如 `double pickup_live_height_cm_; bool has_pickup_live_height_;`），`enterPickupSub(DESCEND_MID)`。
  - 超时（复用 `measure_height_timeout_sec_`）→ **回退**：`has_pickup_live_height_=false`，
    DESCEND 用 `survey_results_[si].height_cm` 兜底，仍 `enterPickupSub(DESCEND_MID)`（不放弃此片）。
- `DESCEND_MID`(cpp:798-808)/`DESCEND_FINAL`(cpp:809-818)：把 `ph` 来源从
  `survey_results_[si].height_cm` 改为"有现场高度用现场、否则用 survey 高度"。
  建议在 enterPickupSub 顶部算一个 `const double ph_use = has_pickup_live_height_ ? pickup_live_height_cm_ : survey_results_[si].height_cm;`
  供 MID/FINAL 用。
- **重试路径**（OBSERVE 失败重试，cpp:977-981）：原来 `enterPickupSub(DESCEND_MID)`，
  改为 `enterPickupSub(MEASURE_HEIGHT_GRAB)`（臂已收回、未夹片，可重新测高）。
- **放置侧不动**：`DESCEND_DROP`(cpp:865-876) 仍用 `empty_pillar_height_cm_`（第一趟测的空柱高）。

### 3.3 第二趟要重新订阅 /height
- 现状：第二趟逻辑不用 `/height`。新方案 MEASURE_HEIGHT_GRAB 要用 `tryStepHeight`，
  它读 `point_height_cm_`（来自 `/height`，回调 `pointHeightCallback`）。确认 `/height` 订阅全程在线
  （`point_height_sub_` hpp:340，应一直订阅；`tryStepHeight` cpp:378 需 `has_point_height_`）。
- `/height` 已加 32767 无效值过滤（笔记第14节）。**夹片飞行时点阵是垃圾值**，但新流程在夹片时
  不调 MEASURE_HEIGHT_GRAB，所以不受影响；仍建议在 MEASURE_HEIGHT_GRAB 进入时 `height_hit_samples_.clear()`
  并只在 `isReached` 后采样（避免移动途中污染，沿用第一趟设计 cpp:691）。

### 3.4 与问题一的耦合（务必同改）
情况(a)抓走后柱顶变成"空黑边白框" = 问题一里**会误检假圆的场景**。
- 若问题一不修：抓成功后 OBSERVE 看到柱顶假圆 → 非 NaN → **误判成失败** → 白白重试/放弃。
- 所以**问题一与问题二必须一起改**，且 OBSERVE 成功判据复用问题一修好的"纯黑实心盘 + 连续N帧"。

## 4. 风险 / 注意

1. **cam_offset 符号未实飞验证**（笔记第6/12节，现 -2.5/0）：第二趟测高同样依赖它把点阵移到柱心，
   若打偏，测高命中率低、超时回退。修视觉时顺带核对点阵实际落点。
2. **classify_circle_size 改排序**（若采纳 1.5）：要同步改 `planPickupOrder`，否则占比与分类两套口径打架。
   保守起见：先只修占比正确性，排序仍按占比降序；分类排序作为可选增强。
3. **连续N帧判失败的 N 取值**：太大→真失败反应慢、太小→抗噪差。建议 N=3~5，参数化放 launch，实飞调。
4. **现场测高耗时**：每片抓取前 settle+测高会增加单片耗时（影响总时间显示）。MEASURE_HEIGHT_GRAB
   超时沿用 `measure_height_timeout_sec_`(5s)，注意总任务时长预算。
5. **本机不编译**：改完在另一台 colcon build；视觉是 Python（visual_pkg/dbh_visual_pkg）改完需重装/重 source
   对应包，detector_core.py 改动随 visual_node 生效。

## 6. 本轮已定决策（用户确认）

- 第二趟测高超时兜底：**回退第一趟 survey 高度**，不放弃该片。

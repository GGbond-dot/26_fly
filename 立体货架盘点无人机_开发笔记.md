# 立体货架盘点无人机（D 题）开发笔记

> 2024 电赛 D 题《立体货架盘点无人机系统》。本分支 `inventory-task` 从 `main` 切出，
> 复用本仓现有飞控链路（PID / uart_to_stm32 / cartographer），只新增"二维码识别 + 激光指示 +
> 盘点任务状态机"。旧版 demo 见 24fly 仓库（GGbond-dot/24fly），核心识别代码已移植本仓。

---

## 0. 一句话方案

单相机 + 空中 yaw 旋转扫面（教练组 2026-05-29 定）：一个相机水平朝前，飞到一个货架面，
逐货位对准→激光点亮 0.5s→读编号→记录；扫完一面整机 yaw 转 180° 扫背面；两个货架四个面共 24 码。
**双相机是后期赶时间的备选**，先把单相机 + yaw 旋转跑通。

---

## 1. 题目要点（评分锚点）

场地 400×500cm，黑方块=起飞点，黑圆=降落点（二者不同位置）。2 个双面货架，每面 6 个货位
（2 行 × 3 列，列距 50cm，行间 40+40cm，二维码 19×19cm，内容为数字 1~24）。

| 要求 | 关键动作 | 分 | 限时 |
|---|---|---|---|
| 1-1 | 垂直起飞到 150±10cm | 5 | — |
| 1-2① | 遍历 24 码，**激光在码范围内点亮 0.5s** | 24 | 270s |
| 1-2② | 地面站 LCD 实时显示 编号+货位(A1~D6)，每盘一个 **LED 亮灭 1s** | 12 | |
| 1-3 | 稳定降落在降落点，几何中心不出界 | 5 | |
| 1-4 | 盘点后地面站显示全部结果，可输入编号查货位 | 5 | |
| 1-5 | 遍历越快越好 | 10 | |
| 2 | 定向盘点：先识别 1 张抽取码报送编号→显示规划航线→飞过去盘点→降落 | 34 | 180s |

硬件硬约束：轴距 ≤45cm；桨全防护；**激光水平安装、不可动**、光斑 ≤6mm；一键启动后全程不得人为干预、不得中途降落。

## 2. 复用 vs 新写（与 24fly 的关系）

24fly 和本仓 26fly 同源，底层 `activity_control_pkg / pid_control_pkg / uart_to_stm32 /
serial_comm / my_carto_pkg / bluesea2` 共享。**26fly 的这些包是进化版（适配 humble + 双USB相机 +
面阵激光的新硬件），不要用旧包覆盖。** D 题只把 24fly 里独有的二维码逻辑搬过来：

| 模块 | 来源 | 处理 |
|---|---|---|
| 二维码识别 + 激光脉冲 + 对准 | 24fly `opencv01/decoder_common.py` | ✅ 已移植为 `qr_vision_pkg`，**改单相机**、加 `rotate_code`、加 `/qr_vision/enable` 门控 |
| 偏移→机体系微调 | 24fly `opencv01/qr_fine_tune.py` | ✅ 已移植，默认前缀改 `/qr_vision` |
| 航点 + 视觉对准切换 | 24fly `activity_control_pkg` 的 `/current_target_camera` 左右切换 | ❌ 不直接用——改成单相机 yaw 航点，逻辑并入新任务节点 |
| 盘点任务状态机 | 新写 | ✅ `inventory_control_pkg/inventory_mission_node`（仿 spray 骨架） |
| 飞控链路 PID/uart/carto | 26fly 现有 | 直接复用，不改 |
| 地面站 LCD/LED + 编号查询 | 新写（待做） | ⏳ 见 §6 |

## 3. 本分支已建骨架（待另一台机 colcon build）

### `src/qr_vision_pkg`（ament_python，移植自 opencv01）
- `qr_vision_pkg/decoder_common.py` → 节点 `qr_vision`：单相机识别。
  发布 `/qr_vision/id`(String)、`/qr_vision/offset_norm`(Point)、`/qr_vision/aligned`(Bool)、
  `/qr_vision/debug_image`。订阅 `/qr_vision/enable`(Bool) 总开关。
  识别到码且横向进窗、且与上次不同码 → 子线程拉低 `laser_pin` 0.5s 打激光。
  关键参数：`camera_device / rotate_code / fourcc=MJPG / laser_pin / eps_x / eps_x_laser /
  stable_frames / decode_interval`。
- `qr_vision_pkg/qr_fine_tune.py` → 节点 `qr_fine_tune`：偏移→机体系 cm 微调（死区+EMA+限步）。

### `src/inventory_control_pkg`（ament_cmake，新写）
- `inventory_mission_node`：盘点任务状态机。
  - 阶段：`IDENTIFY`(仅定向) → `TAKEOFF` → `TRAVERSE` → `RETURN` → `LAND` → `DONE`
  - 航点 `InventoryWaypoint{x,y,z,yaw,scan,slot,tag}`；scan 航点到位悬停开识别，等"对准+读码"。
  - 发布：`/target_position`、`/active_controller(2)`、`/route_choice(1)`、`/qr_vision/enable`、
    `/inventory_result`(String 编号+货位→LCD)、`/inventory_led`(Empty→LED)、
    `/inventory_target`(String 定向目标)、`/mission_complete`。
  - 订阅：`/height`、`/qr_vision/id`、`/qr_vision/aligned`；位姿走 tf `map→laser_link`。
- `launch/inventory_mission.launch.py`：起 qr_vision + qr_fine_tune + 任务节点。
  `mode:=traverse|directed`。**PID/uart/carto 用本仓现有 launch 另起。**

## 4. 任务流（状态机）

**遍历盘点（要求1）**
```
TAKEOFF(home,150) → [A面 A1..A6] → yaw180 → [B面 B1..B6]
                  → 飞货架2 → [C面 C1..C6] → yaw180 → [D面 D1..D6]
                  → RETURN(降落点上空,150) → LAND(降落点)
```
每个 scan 航点：到位 → settle 0.8s → `/qr_vision/enable=true` → 等 `aligned && id`
→ 记录 `slot→编号` → 发 `/inventory_result` + `/inventory_led` → 关识别 → 下一个。
超时 `scan_timeout_sec`(默认6s) 跳过，避免卡死整套。

**定向盘点（要求2）**
```
IDENTIFY(地面识别抽取码→报送编号) → TAKEOFF → 直飞目标货位盘点 → RETURN → LAND
```

## 5. 待标定 / 待决策（动手前必看）

1. **场地坐标系**：`buildTraverseWaypoints()` 里所有 (x,y,z,yaw) 现在是占位示意，
   必须按本机 cartographer 地图原点与题目图1/图2 实测标定。
   重点：起飞点 vs 降落点坐标、两货架红杆中线 x、`scan_standoff_cm`（相机离板面距离，看视场）、
   上/下行高度（图2：下沿 60cm，行间 40+40，码中心约 25cm 和 105cm 量级）。
2. **相机方位**：当前相机安装方位与 24fly 不同 → `rotate_code` 和 `qr_fine_tune` 的
   `invert_body_*` 符号要重新标（决定"看到码偏左/上时往哪飞"）。
3. **激光 GPIO**：`laser_pin` 填实际接线引脚（24fly 用 wPi 10/13）；激光水平安装，
   靠飞机移动让光斑落到码上，识别端 `eps_x_laser` 控制"何时认为对上了才打"。
4. **编号→货位映射（定向盘点核心难点）**：二维码随机张贴，上电时只知道"目标编号"不知货位。
   两条路（`buildDirectedWaypoints` 里留了 TODO）：
   - **先遍历建表**：先飞要求1 把 `inventory_`(货位→编号) 建好，定向时查表得货位再直飞（快但依赖先跑过）；
   - **边飞边读搜索**：定向单独飞一条扫描航线，读到匹配编号即盘点降落（独立但慢）。
   按 180s 限时和比赛流程定。
5. **yaw 旋转的飞控配合**：单相机靠整机 yaw 转 180° 扫背面，确认 PID 的 yaw 通道能稳住转向+悬停。

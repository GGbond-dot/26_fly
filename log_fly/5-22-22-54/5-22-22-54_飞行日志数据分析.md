## 11. 运行日志异常和风险点

### 11.1 启动阶段 TF 暂不可用

启动后约前 12 秒多次出现：

`TF map <- laser_link 失败: "map" ... target_frame does not exist`

Cartographer 在 22:45:41 启动并加入 trajectory 后，任务后续能正常使用位姿。这个问题没有阻断本次任务，但说明 mission 启动早于 map TF 建立。

### 11.2 RViz 崩溃

RViz 在 22:45:49 崩溃：

`rviz2 process has died, exit code -11`

这不影响自动任务主流程，但说明 OrangePi 图形栈/rockchip OpenGL 仍不稳定。正式飞行可以不启动 RViz。

### 11.3 空柱判定与视觉占比矛盾

mission 规划：

`空柱(放置目标)=#2 @ (168.6,-172.0)`

但 survey 中：

`[survey 柱 2] 铁片占比=0.237(有片)`

这意味着：

1. 如果场地确实把最近降落点的柱子设为空柱，则视觉在空柱上误检出了大占比。
2. 如果视觉结果正确，则场地空柱并不是最近降落点那根，硬编码约定失效。
3. 当前代码不会因为视觉发现空柱有片而更换放置目标。

这项必须优先确认，否则即使抓取成功，也可能把铁片放到已有铁片的柱子上。

### 11.4 抓取失败

第一片目标：

`柱 3 @ (76.6,-213.2), 占比=0.292, 柱高=37 cm`

三次都在 `z=61.5 cm` 伸臂吸磁，随后爬升观察，均判定失败。可能原因按优先级：

1. 机械臂/电磁铁没有真实吸到，需查电磁铁供电、接触高度、臂尖位置。
2. `grab_press=2.5 cm` 不够，或 `R=27 cm` 与本次姿态/地面高度不匹配。
3. 视觉对准后下降期间 xy 漂移，臂尖没有落到铁片中心。
4. 抓取成功但观察算法仍看到圆形，误判失败。需要结合现场录像或在抓取观察阶段记录更多视觉状态。

### 11.5 mission 完成后仍有控制数据

mission 在 168.2 s 完成，但 bag 到 214.0 s 才结束，且中间仍有 `/target_velocity` 和 uart “Sent current velocity”日志。应确认 `/mission_complete` 是否实际发布、uart 是否收到并关闭转发。本次 rosbag 没有记录 `/mission_complete` 话题，运行日志也没看到 “Sent mission complete” 类日志。

## 12. 建议下一步

1. 先确认场地：`(168.6,-172.0)` 那根到底是不是空柱。若不是，修改空柱选择逻辑，不能只靠“离降落点最近”。
2. 如果它确实为空柱，回放/现场检查为什么空柱视觉占比达到 `0.237`。
3. 对第一片抓取点 `(76.6,-213.2)` 做定点重复实验：只执行 `center -> descend_mid -> descend_final -> 伸臂吸磁`，观察实际臂尖与铁片中心的 xy 偏差。
4. 抓取阶段临时加日志：进入 `HOVER_GRAB` 前打印当前位姿、面阵高度、视觉偏差最后值、目标 z、柱高、R、grab_press。
5. 核查任务结束路径：确保发布 `/mission_complete`，uart 收到后停止 `/target_velocity` 转发，避免任务完成后继续输出速度。
